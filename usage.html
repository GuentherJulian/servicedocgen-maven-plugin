Usage

Simply add the plugin to the build section of your POM for the module containing services as following. 
  <build>
    <plugins>
      ...
      <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>servicedocgen-maven-plugin</artifactId>
        <version>1.0.0-beta-2</version>
        <executions>
          <execution>
            <goals>
              <goal>generate</goal>
            </goals>
          </execution>
        </executions>
        <configuration>
          <descriptor>
            <info>
              <title>Service Documentation</title>
              <description>This is ...</description>
            </info>
            <host>my.service.com</host>
            <port>8443</port>
            <basePath>myapp/services/rest</basePath>
            <schemes>
              <scheme>https</scheme>
            </schemes>
            <javadocUrl>https://my.service.com/apidocs</javadocUrl>
          </descriptor>
        </configuration>
      </plugin>
    </plugins>
  </build>

Result

After you called mvn package you will find the generated documentation in target/servicedoc/.
